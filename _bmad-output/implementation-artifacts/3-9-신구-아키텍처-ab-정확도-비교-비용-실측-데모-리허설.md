## Story

QA/AI 담당으로서,
신·구 아키텍처를 동일 라벨셋으로 비교하고 게시글당 비용을 실측하여 데모 모드를 결정하기를 원한다,
그래서 "명확 불법(가격·텔레그램·매크로 명시) 10건 실시간 탐지" 데모를 신뢰도 높게 성공시킨다.

> 본 스토리는 **측정·검증·리허설** 스토리다 — 파이프라인 신규 기능은 추가하지 않는다(3-7/3-8에서 완성). single vs agentic을 동일 ground truth로 비교하고, 비용을 실측하고, fast-path 임계를 튜닝하고, 데모를 오프라인 리허설 가능하게 만들고, 기존 알림 시스템이 agentic 탐지와 함께 동작함을 E2E로 확인한다. 데모 당일 아침까지 `DETECTION_MODE` 토글로 모드 선택 가능 상태를 보장한다.

## Acceptance Criteria

**Given** Story 3-5의 `human_label IS NOT NULL` detections와 양 모드(`single` / `agentic`)가 준비된 상태에서
**When** A/B 비교 스크립트를 실행하면

1. 동일 게시글을 양 모드로 재분류하여 `model_version` 분리(`openai:...` vs `agentic:v1:...`)로 DB에 공존시키고, **agreement + Tier별 Recall + Precision + confusion matrix** 비교표를 `docs/`에 기록한다
2. agentic 모드의 게시글당 **평균 비용 ≤ $0.005, p95 ≤ $0.02**를 실측 확인하고(이미지 첨부 유/무 분리 집계), **escalation율**을 보고한다
3. fast-path 임계(`FAST_PATH_CONFIDENCE` 기본 0.80)를 실측 결과로 튜닝한다(권장값 + 근거를 문서화 — 코드 기본값 변경은 선택)
4. `llm_mock.py`에 **에이전트 모드**를 추가하여 오프라인(실 OpenAI 0) 데모 리허설이 가능하다
5. "명확 불법 10건 실시간 탐지" 데모 리허설 스크립트가 작성되고, **single 모드 즉시 회귀 절차**가 문서화된다
6. **[T1 알림 E2E 흡수]** agentic 모드 `tier=T1` 탐지 저장 시 detection이 `notification_events`(PENDING)를 적재하고 백엔드 `NotificationEventProcessor`가 폴링→`minTier=T1` 룰 매칭→채널 발송→`notification_deliveries` 이력 기록까지 **E2E로 확인**한다 (신규 코드 없이 기존 시스템 검증). `minTier=T1` 룰 설정 방법 + 배포 환경변수(`NOTIFICATION_ENCRYPTION_KEY`, `NOTIFICATION_SCHEDULER_ENABLED`)가 문서화된다

### A/B 비교 설계 — 핵심 함정 (Task 1~3)

- **human_label은 post당 1개, 원본 행에만** — 3-5에서 운영자가 라벨한 detection 행(주로 single `openai:...`)에 `human_label`이 붙어 있다. A/B는 **같은 post를 양 모드로 재분류**한 뒤, 각 모드의 `type`을 그 post의 `human_label`(ground truth)과 비교한다. 재분류로 생긴 agentic 행에는 `human_label`이 **없으므로**, `compute_snapshot`처럼 "같은 행의 human_label vs type"을 보면 agentic 쪽이 비교 불가가 된다. → **`{post_id: human_label}` 맵을 따로 만들고 post_id로 조인**하는 것이 정답.
- **agreement vs Recall/Precision 구분**: `compute_snapshot`(L31-75)은 agreement(라벨과 분류가 같은 비율)만 산출 — **재사용 가능**. 그러나 PRD가 요구하는 **Tier별 Recall/Precision/confusion matrix는 신규 계산**(`compute_snapshot`에 없음). type→Tier는 `tier_router.TYPE_TO_TIER` SSOT 재사용.
- **judgeable 분모**: unknown 라벨은 9-type enum과 매칭 불가 → 분모에서 제외(3-5 규율, L40-66). Recall/Precision도 동일하게 unknown 제외.
- **A/B 공존은 무료**: `(post_id, model_version)` UNIQUE + `ON CONFLICT DO NOTHING`(detection_repository.py:7)으로 single `openai:...`와 agentic `agentic:v1:mini+4o:{YYYY-MM}`가 같은 post에 공존. 재실행 멱등.

### 비용 실측 설계 (Task 4)

- **게시글당 비용 출처**: agentic은 `detections.cost_usd`(orchestrator가 전 스테이지 합산, 3-8) 또는 `agent_runs.cost_usd` 합. degrade·실패 trace도 실지출 반영(3-8 리뷰 패치 `AgentResponseError` usage 보존 — 검증 실패 비용도 집계됨).
- **p95는 분포 필요** — 평균만으로 산출 불가. 게시글별 cost_usd 리스트 수집 후 백분위 계산.
- **이미지 유/무 분리**: 해당 post의 `image_urls`/`s3_image_paths` 존재 여부로 그룹화(PRD L78 "이미지 첨부 유/무 분리 집계").
- **비용 모델 기준선**(변경 제안서 L148-161): S1 $0.00042 + S2a $0.0058(escalate 40%) + S3 $0.0080(escalate 전부). escalation 35%에서 평균 ~$0.0040 ✅ ≤$0.005. 3-8 실측 escalate 풀경로 $0.01143 ≤ p95 $0.02. 본 스토리가 표본으로 **재확인**.
